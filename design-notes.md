# 参考サイト分析メモ — 鹿島ノ森から漣へ翻訳する

参考：https://kajimanomori.co.jp/
分析対象：トップページHTML＋`homepage.css` 実コード、各下層ページ概観

このメモの目的は「参考サイトのどこが効いているか」を実装可能な粒度で言語化し、漣（Sazanami）に再構築するときの判断基準にすること。

---

## 1. 技術スタック・配信の特徴

### 採用されている工夫
- **画像はすべてWebP**。PC/SP別ファイル（`fv_slider01.webp` / `fv_slider01_sp.webp`）。
- ヒーロー画像とロゴSVGを `<link rel="preload">` で先読み。`fetchpriority="high"`。
- SP専用の動画（`intro_video_sp.mp4`）あり。SPでは動画、PCでは静止画スライダー。
- ロゴはSVG。
- Google Fontsは `<link rel="preconnect">` で接続最適化。
- jQueryを使用（古典的な実装）。

### 漣で踏襲したい点
- 画像はWebPが基本、PC/SPで別アセット。
- ヒーローはpreloadで初期表示を最適化。
- ロゴはSVGで自前で書く。
- フォントはGoogle Fontsをpreconnect。
- jQuery不要。Vanilla JSで十分。

### 漣で変える点
- 動画の使い方：参考サイトはSPのみ。漣はPC/SPともに**ヒーローに動画を1案として検討**（凪の水面に立つ漣の動画、5〜8秒ループ、無音）。

## 2. タイポグラフィ — フォントと組み

### 参考サイトの実値
```
和文：Zen Old Mincho（Google Fonts）
       → 游明朝 → ヒラギノ明朝 → MS明朝 → serif
欧文：Baskervville（Google Fonts）
       → Times New Roman → Garamond → Georgia → serif
```

### 文字組みの数値（実CSSから抽出）
- **行高：** 1.6 / 1.8 / 2.0 を使い分け（本文は1.8〜2.0が中心）
- **字間：** 0.016em（本文）／ 0.02em（小見出し）／ 0.15em（小キャプス）／ 0.22em（英字大キャプス）

### 漣での方針
- **欧文：Cormorant Garamond** または **Baskervville** を採用。Cormorantは漣の「繊細さ」「水面のゆらぎ」に近い。
- **和文：Zen Old Mincho** を踏襲。明朝体の格と可読性のバランスが現代的。
- 日本語の見出しは**字間0.1em〜0.15em**で広めに、上品さを担保。
- 英文サブは**字間0.2em〜0.25em**でゆったり。
- 行高は本文1.9を基準（参考サイトより少しだけ広く）。

### 大見出しのコンビネーション例（漣で再現したい型）
```
[和文・大]  ひと漣の、贅沢。
[英文・小]  Luxury, in a single ripple.
```
和文を主、英文を従。サイズ比は約 5:1。垂直に並べる。

## 3. カラーパレット

### 参考サイトの実値
- 抽出された色：`#000`、`#101010`、白
- **CSS上には濃い灰しか存在しない**。色彩はすべて写真から来る。

### この設計の意味
- ホテルの色＝そこに行ったときに見える自然の色。
- UIは色を持たない。photographic minimalism。

### 漣での方針
| 用途 | 色 | 値 |
| --- | --- | --- |
| 背景（基本） | 生成りの白 | `#FAF8F4` |
| 背景（純白） | 白 | `#FFFFFF` |
| 文字（基本） | ほぼ黒 | `#1A1A1A` |
| 文字（中間） | 中明度の黒 | `#3A3A3A` |
| 文字（淡） | 罫線・キャプション | `#9A9A9A` |
| アクセント（藍） | 瀬戸内の海 | `#1F3A5F`（ごく僅かしか使わない） |

**ルール：** 写真以外で色を持つのは藍のみ、それも面積1%未満。基本は白×墨。

## 4. レイアウト構造

### ブレークポイント（参考サイトの実体）
```
≤ 767px         — SP
768〜1199px    — タブレット
1200〜1559px   — PC（標準）
1560〜1919px   — PC（大）
≥ 1920px        — PC（特大）
```

### 漣での簡略化
4段階は過剰。3段階に絞る：
```
≤ 767px         — SP
768〜1199px    — タブレット
≥ 1200px        — PC
```
1920px以上は max-width で段抜けを防ぐ程度で十分。

### グリッドの使い方
- **トップページ：** 1カラム縦スクロールが基本。各セクションが画面いっぱいの「章」になる。
- **客室一覧：** カード縦積み（1カラム）で、各カード内で画像とテキストを左右配置。これが「個室を見せる」のに最も向く。グリッドで2〜4カラムにすると逆に安く見える。
- **過ごし方・施設：** 2カラムグリッドや、画像とテキストを互い違いに配置（zigzag）。

### 余白の設計
参考サイトは**セクション間に画面高の40〜80%相当の余白**を取る。これが「静寂」の正体。
漣でも同じ。`section { padding-block: clamp(6rem, 12vw, 12rem); }` 相当を基準に。

## 5. 動き（アニメーション）の設計

### 参考サイトで使われている主要パターン

| パターン | 仕様 | 用途 |
| --- | --- | --- |
| **Fade up** | opacity 0→1, translateY(.5em→0), 0.8s ease-out | 大半のテキスト・画像出現 |
| **Mask reveal** | radial-gradient mask, 3s cubic-bezier(.215,.61,.355,1) | 大見出し、装飾的な見せ場 |
| **Slow zoom** | transform: scale → scale, 10s ease | ヒーロー画像のKen Burns |
| **Scroll indicator** | 円が下方向に消えるパルス, 3s ease-in-out infinite | "Scroll" CTAの脇 |
| **Marquee text** | translateX, 20s linear infinite | 装飾的な英文流れ |

### イージング
**`cubic-bezier(.215, .61, .355, 1)`** が美的な肝。
これは **easeOutCubic** に近く、「ゆっくり止まる」感じ。**漣でも基本イージングはこれ**。

### 漣で実装するアニメーション
1. **基本フェードアップ** — `.8s ease-out` で全要素の出現
2. **ヒーロー画像のスローズーム** — 8〜12s でじわっと拡大、画像切替時にリセット
3. **テキストの行単位フェード** — 大見出しは行ごとに50ms遅延でフェードアップ
4. **画像のマスクリビール** — ページ最初のキービジュアルだけ、上から下へマスクが開く
5. **スクロールインジケーター** — 縦線が下に伸びる→消える、3s ループ

### Intersection Observer の使い方
要素が viewport の **30%入った段階**でアニメーションを起動。早すぎると見えないうちに動き終わる。

### 動きを「やりすぎない」ためのルール
- 1画面に同時に動く要素は最大3つ。
- ホバーで動く量は2〜4pxに留める。
- 1秒以上かかるアニメーションはヒーローと大見出しだけ。

## 6. 写真の使い方

### 参考サイトの基本パターン
1. **フルブリードのヒーロー** — 画面いっぱい、テキストはオーバーレイ
2. **画像＋テキスト左右配置** — セクションコンテンツの定型
3. **スライダー** — 朝食/昼食/夕食など、複数枚を横スワイプで見せる

### アスペクト比
- ヒーロー：**16:9** 〜 **21:9**（横長）
- セクション内：**3:2** または **4:3**
- 客室カード：**3:2**
- 料理：**1:1** または **3:2**（料理は正方形が美味しそう）

### 漣での方針
- ヒーローは**16:9〜21:9**で広めに、上下を切る大胆さ。
- セクション内画像は**3:2**を基本にし、たまに正方形で抜く。
- 縦写真は**1箇所だけ**使う（客室の1枚など）と効果的。

### 画像の演出
- セクション中の画像は**5〜10秒のスローズーム**を継続。動画ではないが動いて見える。
- スクロール時にわずかに**パララックス**（移動量は10〜20%程度、やりすぎると安く見える）。
- 画像とテキストの境界に**余白**を多めに。テキストを画像にかぶせない（参考サイトもそう）。

## 7. ナビゲーション

### 参考サイトの構造
- **PC：** 横並びのテキストリンク。スクロールしても固定。
- **SP：** 右上ハンバーガー。タップで全画面メニュー（フッターレベルの階層まで含む）。
- **言語切替：** 5言語、ヘッダー右に小さくドロップダウン的に。
- **CTA：** 「予約」ボタンが常時ヘッダー右に存在。

### 漣での方針
- PC：横並び固定、ロゴ中央、左に主要ナビ、右に予約CTA、最右に言語切替。
- SP：左ロゴ／右ハンバーガー、メニューは**全画面オーバーレイ**で開く。
- スクロール時：白背景の透過度が100% → 90% に微変化、ヘッダーの高さは少し縮む（72px → 56px）。
- **言語切替：** 漣は当面**JP / EN の2言語だけ**を想定（実装はJPを完成させ、ENはトップだけ用意で良い）。

## 8. ページ構成（参考サイトのサイトマップ）

抽出したリンクから、参考サイトの構成は：

```
/                  トップ
/about/            ホテルについて
/guestrooms/       客室
  /guestrooms/details/   客室詳細（タイプ別）
/restaurant/       レストラン総合
  /restaurant/breakfast/
  /restaurant/lunch/
  /restaurant/dinner/
/lounge/           ラウンジ・バー
/banquet/          宴会・会議
  /banquet/details/
/facilities/       館内施設
/activities/       過ごし方
/access/           アクセス
/news/             お知らせ
  /news/{n}/        記事詳細
/contact/          お問い合わせ
/faq/              よくある質問
```

**Phase 3で漣のサイトマップを作る際の参照点。**

漣ではこの構成をベースにしつつ、規模を圧縮する：
- 宴会・会議は不要（22室の隠れ家には合わない）。
- ハラスメント方針・サイトポリシーは省略可（架空サイトなので）。
- ラウンジは独立ページではなく、レストランのサブセクションに統合。

## 9. 「鹿島ノ森らしさ」の正体（要約）

参考サイトが「効いている」理由を3行に圧縮：

1. **色がない（写真以外は墨と白）** から、写真の中の自然色が際立つ。
2. **動きが遅い（0.8s〜10s）** から、見ている側の呼吸が落ちる。
3. **余白が広い（画面高の50%以上が空白）** から、コンテンツが「語り」になる。

漣でこの3条件を満たせば、世界観の品質は担保される。これが Phase 5（ビジュアル設計）と Phase 6（実装）の北極星。

## 10. 漣で参考サイトを「越える」ポイント

参考サイトは2026年初頭の制作物として優秀だが、いくつかは現代化できる：

- **モダンCSS**：jQuery不要、`@property`、`clamp()` を活用したフルードタイポ。
- **ヒーロー動画**：水面に立つ漣の動画は、静止画スライドより「凪が動く瞬間」を強く伝える。
- **`view-transitions API`**：ページ間遷移をネイティブに滑らかに。
- **`scroll-driven animations`**：Intersection Observer JS不要のスクロール演出（対応ブラウザのみ、フォールバック必須）。
- **`color-mix()`**：色のバリエーションをCSS変数1つから派生。

ただし「全部入れる」は避ける。参考サイトの**抑制の美学**こそが核なので、機能は世界観に貢献する分だけ採用する。
